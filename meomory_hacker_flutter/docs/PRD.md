# Memory Hacker PRD

## Product Summary
Memory Hacker is a mobile-first cyberpunk memory puzzle game built for short, high-focus play sessions in portrait mode. Players act as a digital infiltrator who briefly reveals node patterns, then reconstructs them under pressure before system defenses escalate.

## Problem
Many mobile puzzle games feel disposable, ad-heavy, or visually generic. We want a premium-feeling memory game with stronger atmosphere, sharper UX, and just enough meta-progression to keep sessions sticky without adding production-heavy systems.

## Target Player
- Mobile players who enjoy pattern, memory, and reaction-based puzzle games
- Ages 16 to 40, comfortable with sleek sci-fi aesthetics
- Plays in 2 to 8 minute sessions, often one-handed

## Core Value Proposition
- Fast, satisfying memory puzzles with a strong cyberpunk identity
- Clean premium presentation instead of bloated live-service design
- Easy to learn in seconds, but steadily more demanding through rule twists

## Core Loop
1. Player starts a hack run.
2. A grid pattern is shown for a short reveal window.
3. The board hides and the player reconstructs the pattern.
4. The game scores accuracy, speed, and streak.
5. Player advances, fails, or retries based on performance.

## Design Pillars
- Clarity first: every interaction must read instantly on a phone screen
- Tension through information loss: challenge comes from recall, not input complexity
- Premium feel: polished transitions, typography, sound, and visual feedback
- Solo-dev realism: limited systems, reusable mechanics, handcrafted content

## MVP Features
- Onboarding flow with a short tutorial
- 10 handcrafted levels in one progression path
- Core gameplay using tap-based pattern recall on portrait grids
- Basic fail, retry, pause, and level complete flows
- Level score based on accuracy, time, and attempts
- Local save for unlocked levels and best results
- Lightweight settings for audio, haptics, and accessibility toggles

## Success Criteria
- First session is understandable without external explanation
- Average level duration stays under 90 seconds
- Tutorial completion rate should feel high in manual playtesting
- Replay value comes from score improvement and cleaner execution

## Non-Goals For Launch
- Online multiplayer
- User-generated levels
- Narrative-heavy campaign scenes
- Live events, battle pass, or deep economy systems
- Procedural level generation
